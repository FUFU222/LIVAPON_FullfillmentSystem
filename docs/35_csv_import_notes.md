# CSV インポート機能に関するメモ（2025-11-09）

## 現状
- `/import` ページと Back-end ロジック（CSV バリデーション / エントリ生成）はコードベースに残存。
- 現行の正式運用スコープには含めない。コードや画面が残っていても、現時点の業務要件ではない。
- コンポーネント: `app/import/*`, `components/import/*`（テーブルプレビュー等）。
- API: `app/import/actions.ts` などで preview / validation / `import_logs` 記録まで実装済み。正式な一括登録フローとしては扱わない。

## 現行スコープ外としている理由
- 一部セラーからの利用要望はあるが、CSV 仕様のすり合わせやサポートコストが読めない。
- UI ベースの発送登録と修正依頼フローを優先しており、CSV を正式導線として持たなくても現行要件を満たせる。

## 再開時にやるべきこと
1. 正式運用へ戻すかを先に判断する。
2. CSV テンプレート・ドキュメントを最新版に合わせる（FO 情報や在庫ポリシー変更があれば反映）。
3. Supabase バリデーション（`import_logs` など）やエラーハンドリングを再テストする。
4. 正式な一括登録責務まで持たせるなら、preview と実登録の責務分離を明確にする。

## 今後検討したいこと
- MCP や Bridge App からの一括登録 API 化（CSV 不要にできるか）。
- セラー別のカスタムフォーマットを許容する場合の変換ロジック。
- 監視/通知: `import_logs` を UI で確認できる画面を整備すると運用がしやすい。

> ※機能自体は残っているが、現時点では「業務上不足している機能」ではなく「保留している機能」として扱う。
