# CSV インポート機能に関するメモ（2025-11-09）

## 現状
- `/import` ページと Back-end ロジック（CSV バリデーション / エントリ生成）はコードベースに残存。
- ヘッダーのナビゲーションからはリンクを一旦外し、ユーザーからの導線は閉じた状態。
- コンポーネント: `app/import/*`, `components/import/*`（テーブルプレビュー等）。
- API: `app/import/actions.ts` などで Supabase への一括登録を実装済み。

## 需要不確実のため非表示にした理由
- 一部ベンダーからの利用要望はあるが、CSV 仕様のすり合わせやサポートコストが読めない。
- OM モデル移行や FO PULL 実装など優先度の高いタスクがあるため、UI の露出を控えている。

## 再開時にやるべきこと
1. `/import` メニューをヘッダーに戻し、権限チェックを再確認（ベンダーのみアクセス可能に）。
2. CSV テンプレート・ドキュメントを最新版に合わせる（FO 情報や在庫ポリシー変更があれば反映）。
3. Supabase バリデーション（`import_logs` など）やエラーハンドリングを再テスト。
4. ユーザー教育：CSV ヘルプを docs に追記し、操作方法を案内。

## 今後検討したいこと
- MCP や Bridge App からの一括登録 API 化（CSV 不要にできるか）。
- ベンダー別のカスタムフォーマットを許容する場合の変換ロジック。
- 監視/通知: `import_logs` を UI で確認できる画面を整備すると運用がしやすい。

> ※機能自体は残っているので、再需要があればすぐ再公開可能。
